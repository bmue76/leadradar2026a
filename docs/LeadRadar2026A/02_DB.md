## Lokale Dev-DB (empfohlen): PostgreSQL auf Port 5433 (known creds)

### Warum 5433?
Auf vielen Windows-Setups läuft bereits ein PostgreSQL Windows-Service auf **5432** – oft mit unbekanntem `postgres` Passwort.
Damit Dev-Setup reproduzierbar ist, nutzen wir eine separate Dev-Instanz auf **5433** mit bekannten Credentials:

- User: `postgres`
- Passwort: `postgres`
- Port: `5433`

Beispiel-URL:
```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5433/leadradar2026a?schema=public"
